# Part 1｜專案總覽

## 1.1 專案目的

這是 **花蓮高商（國立花蓮高級商業職業學校）二年級彈性課程「多元文化與文學」** 的原住民族單元成果作業。

目的有兩個：

1. **教學目的**：讓學生透過浦忠成〈縱谷無言〉一文與《zalan 見識南島》影片，認識花蓮在地四個原住民族重大歷史事件，並完成一份三頁式數位報告。
2. **觀課目的**：這個單元是原住民族校定課程計畫的觀課成果關鍵之一，最終產出（班級成果網頁 + 事件地圖視覺）將作為觀課展示用。

## 1.2 學生情境（設計時請以此為準）

- **年級**：花蓮高商二年級（職業學校）
- **動機程度**：偏低，傾向不帶回家做作業，所以三節課內必須完成所有事
- **程度差異**：明顯不一，有人查資料就卡關，有人寫得出深度省思
- **科技程度**：能用 Google 文件、能上網,但不熟 AI 工具，需要明確 SOP

## 1.3 設計原則（不可違反）

1. **降低操作門檻**：工具只有兩個（Google 文件 + NotebookLM），不再增加
2. **結構化引導**：學生不需要決定版面，只需要填內容
3. **限定 AI 角色**：AI 只能用來「幫助理解」，不可代寫
4. **正確度優先於美觀**：寧可看起來樸素，也不要資訊錯誤
5. **產出必須觀課可展示**：成果要能掛在網頁、印出來貼牆

## 1.4 三節課的時程

| 節次 | 學生在做什麼 | 工具 |
|------|------|------|
| 第一節 | 認識作業、選定事件、填第一頁（基本事實 + 關鍵照片） | Google 文件 + 指定網路來源 |
| 第二節 | 用 NotebookLM 深化理解、填第二頁（多視角對照、地景現況） | NotebookLM + Google 文件 |
| 第三節 | 填第三頁（個人省思）、互看 2-3 份同學作品、繳交 | Google 文件 |

## 1.5 老師（Lai）的工作分工

**課前要做（人工）**：
- 為四個事件分別建立 NotebookLM 的 Notebook，預先上傳指定資料
- 在 Google 教室派發 Google 文件模板（由 Claude Code 產生 HTML 預覽稿，老師據此手動建立 Google 文件版本）
- 列印給學生的「指定網址清單」紙本（避免學生迷失在搜尋頁）

**課中要做**：
- 第一節示範如何使用模板、如何用指定網址查資料
- 第二節示範 NotebookLM 與三種 AI 提問範本
- 第三節主持互評、檢查繳交

**課後要做（人工 + Claude Code）**：
- 收齊所有學生 Google 文件，下載打包
- 把打包檔案丟給 Claude Code，由 Claude Code 自動生成「班級成果網頁」與「事件地圖視覺」

# Part 2｜學生成品規格：三頁式報告

## 2.1 整體規格

- **格式**：Google 文件
- **頁數**：A4 三頁，每頁有明確主題
- **設計**：用表格切割版面，學生只填內容、不可改版型
- **字型**：標楷體（內文）+ 微軟正黑體（標題）
- **每頁規劃**：
  - 第一頁：基本事實（What & Where）
  - 第二頁：深化理解（Who's speaking & What's left）
  - 第三頁：個人省思（So what）

## 2.2 第一頁：基本事實

```
標題：縱谷無言．原住民族歷史事件研究
副標題：多元文化與文學．原住民族單元

[班級|座號|姓名 表格]

我選擇的事件（勾選一個）：
□ 大港口事件（1877）
□ 大分事件（1915）
□ 七腳川事件（1908）
□ 太魯閣戰役（1914）

──────────────────────────────────
一、事件基本資料（每格 30-50 字）
   時間：
   地點（古地名 / 今地名）：
   涉及族群：
   主要人物（至少兩位）：

──────────────────────────────────
二、事件經過（150-200 字）
   引導句：用你自己的話說，發生了什麼事？
   [作答區三行]

──────────────────────────────────
三、關鍵照片或圖像（至少一張）
   要附：一張可以代表這個事件的圖片
   要寫：圖片標題、來源網址、為什麼選這張
   [圖片插入區]
   圖說：
   來源：
   我為什麼選這張：
```

## 2.3 第二頁：深化理解

```
四、不同視角的對照（NotebookLM 任務）
   引導：同一件事，不同人講出來不一樣

   A. 官方記載（清廷／日本）怎麼說？（80字）
   B. 部落族人後代怎麼說？（80字）
   C. 學者（如蔡中涵、浦忠成）怎麼說？（80字）
   D. 三個版本最不一樣的地方是？（50字）

──────────────────────────────────
五、今天留下了什麼？（地景現況）
   引導：這個事件現在還看得到痕跡嗎？

   勾選類型（可複選）：
   ▢ 紀念碑（在哪裡？什麼樣子？）
   ▢ 地名（古名變成什麼？或保留原名？）
   ▢ 學校／建築（蓋在哪？覆蓋了什麼？）
   ▢ 紀念活動（每年有沒有人記得？）
   ▢ 完全沒留下（為什麼？）

   我的觀察（120-150 字）：

   要附：一張現在的照片（紀念碑、地景、活動）
   [圖片插入區]
   圖說：
   來源：
```

## 2.4 第三頁：個人省思 + 資料來源

```
六、回到浦忠成（80-100 字）
   引導：重新讀〈縱谷無言〉提到這個事件的段落
        （如果文章沒提到，思考為什麼沒提到）
   填空起點：「讀完事件後再讀文章，我發現＿＿」

──────────────────────────────────
七、個人省思（半填空，3 題）
   引導：不需要寫得華麗，誠實寫下來就好

   1. 讀完這個事件，最讓我意外的是＿＿＿，
      因為＿＿＿（80字）

   2. 如果我是這個事件當事人的後代，我會＿＿＿（80字）

   3. 以後我搭火車經過花蓮這個地方時，我會想起＿＿＿（80字）

──────────────────────────────────
八、資料來源（至少 3 筆，含 1 筆 NotebookLM）

   1. 標題：
      網址：
      類型：□ 官方 □ 學術 □ 媒體 □ 部落

   2. 標題：
      網址：
      類型：□ 官方 □ 學術 □ 媒體 □ 部落

   3. 標題：
      網址：
      類型：□ 官方 □ 學術 □ 媒體 □ 部落

──────────────────────────────────
九、互評欄（同學填寫，2 句即可）
   同學 A 看了我的報告後說：
   同學 B 看了我的報告後說：
```

## 2.5 為什麼這樣設計

- **半填空式省思**：動機低落學生看到「請寫省思」會放棄，看到「最讓我意外的是＿＿，因為＿＿」就寫得出來
- **多視角對照**：強迫學生使用 NotebookLM，不會的話這格寫不出來
- **資料類型核選**：訓練學生分辨資料來源差異，而不是隨便貼連結
- **互評欄**：第三節的活動，給同學留言而不是上台報告

老師需要建立 **四個 Notebook**，每個事件一個。每個 Notebook 上傳的資料如下（詳細網址見 Part 4）：

6. 公視原民台的影音報導文字稿（如有）

**Notebook 命名規則**：`花商二年級_[事件名稱]`
範例：`花商二年級_大港口事件`

## 3.2 學生使用 SOP

### 第一節課（45 分鐘）

```
[5分] 老師示範：開啟 Google 文件模板（已派發到 Google 教室）
       → 學生「建立副本」存到自己的雲端
       → 填寫班級座號姓名

[5分] 老師說明四個事件，學生勾選一個
       提示：「沒有對錯，憑直覺選你最好奇的那個」

[5分] 老師示範：打開「指定網址清單」
       → 從第一個連結開始查
       → 強調：不准開 YouTube、不准滑社群

[25分] 學生自行填寫第一頁
       → 基本資料、事件經過、關鍵照片

[5分] 收尾：老師檢查每位學生第一頁完成度
       → 沒做完的記錄下來，下節課優先處理
```

### 第二節課（45 分鐘）

```
[5分] 老師示範：打開 NotebookLM
       → 進入老師建好的對應 Notebook
       → 強調：只能用這四個 Notebook，不要自己另開

[10分] 老師示範三種 AI 提問範本（重要！）

       提問範本 1（理解）：
       「請用高中生能懂的話，整理[事件名]中官方
        記載與部落觀點的差異，各 80 字。」

       提問範本 2（整理）：
       「[事件名]這個事件，現在在地景上還留下
        什麼痕跡？請整理出至少三點。」

       提問範本 3（省思引導）：
       「如果我是[事件名]當事人的後代，現在看著
        這個地方，我可能會有什麼感受？請列三種
        可能的感受。」

       強調：不可以問「幫我寫一段 200 字的省思」
       那是代寫，不是學習。

[25分] 學生自行填寫第二頁
       → 多視角對照、地景現況觀察

[5分] 收尾：抽問 3-5 位學生，請他唸出自己寫的
       「三個版本最不一樣的地方」
```

### 第三節課（45 分鐘）

```
[15分] 學生填寫第三頁：個人省思
       → 半填空題，不准空白

[15分] 互評時間
       → 學生兩兩交換看（換 2 次，看到 2 位同學的）
       → 在「互評欄」留下 1-2 句感想
       → 規則：只能寫優點與問題，不可寫批評

[10分] 補充修正、確認資料來源欄
       → 沒填滿 3 筆來源的補上

[5分] 全班一起按下「分享 → 老師」
       → 老師當場確認收到所有檔案
```

## 5.1 視覺風格指南：「在地族群識別 + 編輯型策展」

這是 Lai 老師指定的風格。設計過程經過數輪討論後定型，**不是**單一誠品風或純文人雅士風，而是融合以下三個來源：

### 整體調性

**不是**：
- 學校公告風、Word 預設樣式、PowerPoint 模板感
- 純粹文人雅士風（過度留白、無內容、太老氣）

- 淺深兩種模式，因應不同使用情境

### 雙模式設計（重要！）

- 高度 4px

- 不要圓角（單側邊框搭圓角會很怪）

- 署名用對應事件主色、字距 0.15em

### 排版原則

- **留白優先**：行高 1.8-2.0、區塊間距 2-3rem
- **單欄為主**：除非雙欄並列卡片，否則不分欄
- **小標籤用字距創造節奏**：所有 11px 小標籤都設 letter-spacing: 0.3em 以上
- **避免邊框**：用背景色塊與留白做區隔，不靠 border
- **避免裝飾**：不加陰影、漸層、emoji、霓虹效果
- **圖片處理**：老照片可加微 sepia 濾鏡（5-10%）營造歷史感

## 6.3 設計守則（不可違反）

1. **資料與表現分離**：所有可變資料放 JSON，HTML 只負責渲染
2. **不要硬編碼學生資料**：學生作品永遠從 JSON 讀取
3. **不要過度設計**：簡單比花俏更符合本專案調性
4. **不要自行新增工具**：只用 Google 文件 + NotebookLM
5. **不要修改視覺風格**：配色、字型、四色事件分色已寫死，請嚴格遵守
6. **不要產生需要 build 流程的程式碼**：Lai 老師沒有 React/Vite 環境，所有 HTML 應為單檔可開
7. **JS 框架限制**：盡量純 HTML+CSS+vanilla JS；如需 framework，僅可用 CDN 的 Alpine.js 或 React UMD（不可 build）
8. **遵循 KISS／DRY／YAGNI／SOLID**：穩定優先於設計，先求功能正確再求美觀

### 雙模式對應表（重要！）

每個檔案必須使用對應的模式，不可弄錯：

| 檔案 | 模式 | 理由 |
|------|------|------|
| `student_report_template.html` | 淺色 | 學生會列印或螢幕長時間閱讀 |
| `student_sop_poster.html` | 淺色 | 印出貼牆 |
| `student_resources_handout.html` | 淺色 | 紙本印刷 |
| `class_showcase.html` | **深色** | 觀課投影、劇場感 |
| `event_map_interactive.html` | **深色** | 嵌入網頁、與成果網頁一致 |
| `event_map_print.svg` / `.pdf` | 淺色 | 印刷貼牆 |

### 共用設計 Token 必須抽出成 CSS 變數

請在 `assets/styles/design_tokens.css` 中定義所有顏色與字型，所有 HTML 透過變數引用。淺深兩模式可用 CSS class（如 `.theme-light` / `.theme-dark`）切換。範例：

}

.theme-light {
  --bg-main: #F5F1EA;

  /* ... */
